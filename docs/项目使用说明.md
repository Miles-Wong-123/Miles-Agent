# Miles-Agent 项目使用说明

这份文档面向“把项目跑起来并完成一次完整使用”这个目标，尽量只保留最常用的信息。

## 目录结构

- `server/`：Spring Boot 后端
- `web/`：Vue 3 + Vite 前端
- `start-miles-agent.sh`：一键启动脚本
- `stop-miles-agent.sh`：一键停止脚本

## 运行前准备

本项目依赖以下环境：

- JDK 17
- Node.js
- pnpm
- Docker

还需要准备以下服务或配置：

- Redis
- PostgreSQL + pgvector
- DashScope API Key
- BigModel API Key
- 发件邮箱 SMTP 账号和授权码

## 本地配置

首次运行前，先复制本地配置文件：

```bash
cp server/src/main/resources/application-local.example.yml server/src/main/resources/application-local.yml
```

然后在 `server/src/main/resources/application-local.yml` 中填写你自己的配置。

重点字段：

- `spring.mail.username`
- `spring.mail.password`
- `spring.data.redis.host`
- `spring.data.redis.port`
- `spring.data.redis.password`
- `pgvector.host`
- `pgvector.port`
- `pgvector.database`
- `pgvector.user`
- `pgvector.password`
- `langchain4j.community.dashscope.*.api-key`
- `bigmodel.api-key`

## 一键启动

仓库根目录已经准备好了启动脚本，推荐优先使用：

```bash
cd /Users/wmy/Code/Miles-Agent
./start-miles-agent.sh
```

这个脚本会自动执行这些动作：

- 启动 `miles-agent-pgvector` 容器
- 启动 `miles-agent-redis` 容器
- 等待 `5432` 和 `6380` 端口就绪
- 自动构建 `web/` 前端静态资源
- 进入 `server/` 目录执行 `./mvnw spring-boot:run`

注意：

- 脚本最后是前台启动 Spring Boot，所以终端会持续输出日志
- 这不是卡住，而是服务正在运行
- 测试接口时，请另外开一个新终端窗口

如果脚本提示前端依赖缺失，先执行一次：

```bash
cd /Users/wmy/Code/Miles-Agent/web
pnpm install
```

## 停止项目

停止项目和相关中间件：

```bash
cd /Users/wmy/Code/Miles-Agent
./stop-miles-agent.sh
```

这个脚本会：

- 关闭占用 `10010` 端口的 Java 进程
- 停止 `miles-agent-redis`
- 停止 `miles-agent-pgvector`

## 单独启动后端

如果你不想走一键脚本，也可以单独启动后端：

```bash
cd /Users/wmy/Code/Miles-Agent/server
./mvnw spring-boot:run
```

前提是：

- Redis 已经可用
- pgvector 已经可用
- `application-local.yml` 已经填写正确

## 单独启动前端开发环境

前端开发调试时使用：

```bash
cd /Users/wmy/Code/Miles-Agent/web
pnpm install

```

常用命令：

```bash
pnpm test
```

说明：

- `pnpm build` 会把前端产物输出到 `server/src/main/resources/static/`

## 服务地址

默认情况下：

- 前端页面：`http://127.0.0.1:10010/api/`
- 登录页：`http://127.0.0.1:10010/api/login`
- 注册页：`http://127.0.0.1:10010/api/register`
- 健康检查：`http://127.0.0.1:10010/api/actuator/health`
- Prometheus 指标：`http://127.0.0.1:10010/api/actuator/prometheus`

建议调试时优先使用 `127.0.0.1`，比 `localhost` 更稳一些。

## 如何判断服务是否启动成功

### 1. 看健康检查

```bash
curl http://127.0.0.1:10010/api/actuator/health
```

正常会返回包含 `"status":"UP"` 的 JSON。

### 2. 看端口是否监听

```bash
lsof -nP -iTCP:10010 -sTCP:LISTEN
```

### 3. 看 Docker 容器是否在线

```bash
docker ps
```

正常应该能看到：

- `miles-agent-pgvector`
- `miles-agent-redis`

## 首次使用

### 浏览器方式

1. 打开 `http://127.0.0.1:10010/api/`
2. 如果未登录，页面会自动跳到 `/login`
3. 没有账号就进入 `/register`
4. 输入邮箱并接收验证码
5. 验证码通过后填写昵称和密码
6. 注册成功后会自动登录并进入聊天页

### API 方式

如果你想直接用 `curl` 调业务接口，需要先登录拿 cookie。

登录示例：

```bash
curl -i -c /tmp/miles-agent.cookie -b /tmp/miles-agent.cookie \
  -X POST http://127.0.0.1:10010/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "your-email@example.com",
    "password": "Pass12345"
  }'
```

之后所有受保护接口都要带上：

```bash
-b /tmp/miles-agent.cookie
```

## 接口测试

### 1. 登录态探活

```bash
curl -i -b /tmp/miles-agent.cookie http://127.0.0.1:10010/api/auth/me
```

未登录时会返回 `401`，已登录时会返回当前用户信息。

### 2. 测试聊天接口

```bash
curl -i -b /tmp/miles-agent.cookie \
  -X POST http://127.0.0.1:10010/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "sessionId": "session-2002",
    "prompt": "你好，介绍一下你自己"
  }'
```

建议：

- 每次调试尽量换一个新的 `sessionId`
- 这样可以避免旧的 Redis 会话记忆影响当前回答

### 3. 测试流式接口

```bash
curl -N -b /tmp/miles-agent.cookie \
  -X POST http://127.0.0.1:10010/api/streamChat \
  -H "Content-Type: application/json" \
  -d '{
    "sessionId": "session-3001",
    "prompt": "请用一句话介绍这个项目"
  }'
```

### 4. 测试知识写入接口

```bash
curl -i -b /tmp/miles-agent.cookie \
  -X POST http://127.0.0.1:10010/api/insert \
  -H "Content-Type: application/json" \
  -d '{
    "question": "这个项目是做什么的？",
    "answer": "这是一个基于 Spring Boot 和 LangChain4j 的 AI Agent 项目。",
    "sourceName": "MilesAgent.md"
  }'
```

### 5. 测试匿名访问是否被拦截

```bash
curl -i -X POST http://127.0.0.1:10010/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "sessionId": "session-anon",
    "prompt": "hello"
  }'
```

正常会返回 `401`，这是预期行为。

## 常见问题

### 1. 启动时报数据库连接失败

常见现象：

- `Connection to localhost:5432 refused`
- `BeanCreationException`

排查方式：

```bash
docker ps
lsof -nP -iTCP:5432 -sTCP:LISTEN
```

如果没起来，先执行：

```bash
docker start miles-agent-pgvector
```

### 2. 启动时报 Redis 连接失败

排查方式：

```bash
lsof -nP -iTCP:6380 -sTCP:LISTEN
docker ps
```

如果没起来，先执行：

```bash
docker start miles-agent-redis
```

### 3. 聊天接口返回 401

这是正常现象，说明你还没有登录，或者请求里没有带 session cookie。

排查方式：

```bash
curl -i http://127.0.0.1:10010/api/auth/me
```

解决方式：

- 先在浏览器里注册/登录
- 或先调用 `/api/auth/login`，再带着 cookie 调 `/api/chat`

### 4. 运行 `start-miles-agent.sh` 后终端一直不返回

这是正常现象。

原因：

- 脚本最后以前台方式运行 `./mvnw spring-boot:run`
- 所以终端会持续显示后端日志

处理方式：

- 保持这个窗口不动
- 另开一个新终端做 `curl` 测试或前端调试
